# HUNT - Research and Information Gathering

## Overview
The HUNT skill provides a systematic approach to research and information gathering. It helps you find, evaluate, and organize information efficiently to answer questions and support decision-making.

## Workflow

### 1. Question Formulation
- Clearly define what you need to find out
- Break complex questions into specific, answerable sub-questions
- Identify what type of answer is needed: factual, opinion, comparative, or procedural
- Determine the required depth: overview vs. deep technical detail
- Note any time sensitivity: does the information need to be current?

### 2. Source Identification
- Identify primary sources: official documentation, source code, specifications
- Identify secondary sources: tutorials, blog posts, Stack Overflow, forums
- Identify tertiary sources: summaries, comparison articles, Wikipedia
- Prioritize sources by reliability: official docs > peer-reviewed > community > opinion
- Check the date of sources; prefer recent information for fast-moving topics

### 3. Search Strategy
- Use specific, targeted search queries; avoid overly broad terms
- Try multiple query formulations to capture different angles
- Use advanced search operators when available (site:, filetype:, intitle:)
- Search in multiple contexts: web, GitHub issues, documentation, academic sources
- Keep track of what you have already searched and what you found

### 4. Information Extraction
- Extract the specific information that answers your sub-questions
- Note the source and date for every piece of information gathered
- Distinguish between facts, opinions, and speculation in the sources
- Capture code examples, commands, or configurations with their context
- Note any contradictions between sources and flag them for resolution

### 5. Verification
- Cross-reference findings across multiple independent sources
- Verify technical claims by testing in a controlled environment when possible
- Check that information is current and not deprecated
- Look for consensus among authoritative sources
- Note any information that could not be verified

### 6. Synthesis and Organization
- Organize findings by sub-question or topic area
- Create a summary that directly answers the original question
- Include relevant details, code snippets, and references
- Note remaining gaps or areas requiring further investigation
- Present findings in a clear, structured format

## Best Practices
- Always cite your sources; include URLs and access dates
- Be skeptical of single-source claims; seek corroboration
- Prefer primary sources over interpretations whenever possible
- Check GitHub issues for real-world problems not covered in documentation
- Look at commit history and release notes for the most current information
- Save important pages (archive, screenshot) in case links go stale
- Document dead ends; they save future researchers from repeating them

## Research Checklist
- [ ] Question clearly defined and broken into sub-questions
- [ ] Multiple source types consulted (docs, code, community)
- [ ] Information verified across independent sources
- [ ] Currency of information confirmed
- [ ] Contradictions identified and noted
- [ ] Findings organized and summarized
- [ ] Sources documented with URLs
- [ ] Gaps and next steps identified
