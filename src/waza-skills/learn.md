# LEARN - Learning and Knowledge Synthesis

## Overview
The LEARN skill provides a structured approach to acquiring new knowledge, understanding complex topics, and synthesizing information into actionable understanding. It is designed for efficient, deep learning.

## Workflow

### 1. Learning Objective Definition
- Define what you want to be able to do after learning (not just "understand")
- Set a specific, measurable outcome: "build X", "explain Y", "implement Z"
- Identify the scope: breadth-first survey or depth-first mastery?
- Determine the time budget and adjust scope accordingly
- Assess prerequisite knowledge; identify gaps that need filling first

### 2. Knowledge Mapping
- Create a concept map of the topic: key terms, relationships, hierarchy
- Identify foundational concepts that must be understood first
- Map the learning path: what order should concepts be learned?
- Find the canonical resources: official docs, definitive books, authoritative courses
- Identify the 20% of concepts that explain 80% of the topic (Pareto principle)

### 3. Active Learning
- Study one concept at a time; do not jump around
- After reading, close the material and explain it from memory
- Write summaries in your own words; do not copy-paste
- Create examples that exercise the concept in a novel way
- Teach the concept to someone (or pretend to); teaching reveals gaps

### 4. Practical Application
- Build something small that uses the new knowledge
- Start with a tutorial, then modify it to make it your own
- Intentionally break things to understand error modes and edge cases
- Compare your implementation with established patterns or reference implementations
- Iterate: the first attempt teaches the basics, the second teaches the nuances

### 5. Synthesis and Integration
- Connect new knowledge to existing knowledge: how does this relate to what you know?
- Identify patterns and principles that generalize across domains
- Note contradictions with prior understanding; resolve them
- Create a personal reference: cheatsheet, notes, or mental model
- Identify when to use this knowledge and when not to (boundaries of applicability)

### 6. Retention Planning
- Schedule spaced repetition for critical concepts you need to retain
- Create flashcards or summary notes for key concepts
- Plan periodic review: re-read notes after 1 day, 1 week, 1 month
- Apply the knowledge regularly; use it or lose it
- Update your mental models as you encounter new information

## Best Practices
- Learning by doing is always faster and deeper than passive reading
- Struggle productively: spend real time on hard problems before seeking solutions
- Focus on understanding principles, not memorizing syntax (syntax can be looked up)
- Build a second brain: maintain notes that you can reference later
- Learn in focused sessions (45-90 min) with breaks between
- Embrace confusion; it is a sign you are at the edge of your understanding
- Compare multiple explanations of the same concept; different perspectives unlock understanding

## Knowledge Capture Template
- **Concept**: [Name]
- **Definition**: [In your own words, 1-2 sentences]
- **Why it matters**: [What problem does it solve?]
- **Example**: [Minimal working example]
- **Related concepts**: [Links to connected knowledge]
- **Common pitfalls**: [What trips people up?]
- **When to use / not use**: [Boundaries of applicability]
